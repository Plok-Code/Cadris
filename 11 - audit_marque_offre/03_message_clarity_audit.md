# 03_message_clarity_audit

## Clarté globale
**Bonne.**

Le message maître est net :
Cadris donne à un projet construit avec l’IA un **cadre clair** pour avancer sans dériver.

Le discours est crédible parce qu’il reste mesuré :
- pas de build autonome fort ;
- pas de promesse de vitesse comme centre ;
- pas de posture messianique ;
- pas de dilution dans une simple logique documentaire.

## Messages trop vagues
- “cohérence” quand le mot n’est pas relié à une situation d’usage ;
- “tenir dans le temps” si rien ne montre ce que cela change concrètement ;
- “source de vérité” si elle n’est pas ancrée dans les décisions, le périmètre, la logique et les arbitrages.

## Messages trop ambitieux
À éviter :
- toute lecture “copilote total” ;
- toute lecture “build votre produit à votre place” ;
- toute lecture “plus besoin de réfléchir” ;
- toute lecture “gouvernance complète” en V1.

## Preuves insuffisantes
Les preuves sont conceptuellement solides, mais il manque encore des preuves démonstratives plus visibles :
- un avant / après ;
- un exemple de ce qui fait foi ;
- une scène claire de modification ou de handoff ;
- une démonstration de la différence entre projet qui avance et projet qui tient.

## Formulations à retravailler
### À garder
- cadre clair
- dérive du projet
- ce qui fait foi
- source de vérité
- mémoire de décision
- base exploitable
- projet transmissible
- avancer sans dériver

### À mieux encadrer
- dossier canonique
- continuité légère
- tenue du projet
- projet SaaS

### À éviter
- app builder
- générateur de documentation
- copilote total
- build autonome
- productivité infinie
- wrapper IA

## Conclusion
Le messaging est suffisamment clair pour une V1, à condition de garder une hiérarchie ferme :
- **façade** : cadre clair
- **tension** : dérive
- **preuve de profondeur** : source de vérité
- **objet produit** : base de cohérence exploitable
