# Business rules

## Regles canoniques

- une seule mission active de cadrage par projet au MVP
- une mission a exactement un contexte d'entree
- la mission `Demarrage` est le seul flow MVP
- une question active a la fois dans la premiere tranche
- le dossier n'est disponible qu'apres reponse et reprise reussie
- le markdown n'est jamais la source de verite

## Regles d'etat

- `waiting_user` signifie qu'une action utilisateur est requise
- `completed` signifie que la mission a stabilise un premier dossier
- `ready_to_decide` signifie que le bloc est assez solide pour avancer, pas qu'il est parfait

## Regles de transmission

- toute decision importante doit se retrouver dans le canonique
- un artefact ne peut pas etre considere final s'il ne provient que du chat libre
