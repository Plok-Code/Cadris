# Dossier canonique Cadris

## But

Ce dossier sert de base unique pour coder Cadris sans reposer des questions de cadrage deja tranchees.

Il ne remplace pas les `docs initiaux`.
Il les consolide en version plus directe pour :
- un agent de code ;
- un lead dev ;
- une equipe qui doit lancer le build sans reconstruire le projet mentalement.

## Regle de lecture

Si une ambiguite apparait :
- appliquer l'ordre d'autorite ;
- appliquer les decisions par defaut ;
- ne pas rouvrir une decision deja close sans raison technique forte ;
- documenter tout ecart justifie dans le `decision log`.

## Portee

Ce dossier couvre :
- strategie produit ;
- cadrage MVP ;
- UX/UI utile au build ;
- architecture et contrats ;
- execution, tests et risques ;
- handoff final pour agent de code.

Il ne cherche pas a etre exhaustif sur :
- le legal contractuel detaille ;
- le go-to-market complet ;
- les operations post-scale.
