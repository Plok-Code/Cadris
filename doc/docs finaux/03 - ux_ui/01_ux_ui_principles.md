# UX/UI principles

## Principe directeur

Cadris doit donner le sentiment qu'une mission avance serieusement sans transformer l'utilisateur en chef d'orchestre d'un systeme opaque.

## Lois d'interface

- concret avant taxonomie
- synthese d'abord
- une decision a la fois
- friction utile conservee
- progression perceptible
- incertitude visible
- reprise sans fatigue

## Marque dans l'interface

La marque reste :
- sobre
- structurelle
- secondaire par rapport a la comprehension

Le logo ne doit jamais remplacer un libelle utile.

## Regles de lisibilite

- une seule zone dominante par ecran
- corps principal sobre
- labels simples
- statuts verbaux
- bordures avant ombres
- accent petrol rare
