# Business model and pricing

## Modele economique retenu

Le projet converge vers un modele hybride :
- entree simple sur une mission ou un volume de travail resserre
- puis valeur recurrente si le projet enchaine cadrage, revision et nouveaux dossiers

## Ce qui est monnayable

- nombre de missions
- profondeur du dossier
- niveau de revue ou de premium
- historique / revision / export

## Hypothese de packaging

- plan `Core`
- plan `Extended`
- plan `Operating Layer`

## Regle produit utile pour le build

Le pricing ne doit jamais dicter une complexite produit V1 artificielle.
Le code MVP doit donc rester compatible avec :
- compte simple
- quota simple
- gating fonctionnel simple

Pas de grille tarifaire complexe a encoder des la premiere tranche.
