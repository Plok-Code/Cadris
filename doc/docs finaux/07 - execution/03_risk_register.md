# Risk register

## Risques critiques

### R-01
Le runtime et le control-plane se recouvrent mal.

Mitigation :
- frontieres de service strictes

### R-02
Le projet glisse vers un chatbot ou un cockpit.

Mitigation :
- UI mission-first
- une question a la fois

### R-03
Le canonique derive vers du markdown libre.

Mitigation :
- graphe ou tables canoniques avant export

### R-04
Le scope grossit avant preuve de valeur.

Mitigation :
- gel du flow `Demarrage`

## Risques moyens

- auth provider final pas encore integre
- runtime OpenAI / Restate non branche en premier lot
- share links et PDF repousses
