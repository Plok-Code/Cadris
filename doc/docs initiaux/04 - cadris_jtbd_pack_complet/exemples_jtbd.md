# Exemples de JTBD

## Bon exemple 1
Quand je dois suivre plusieurs demandes clients en parallèle, je veux voir leur état réel en un seul endroit, afin de ne pas oublier de réponse importante.

Pourquoi c'est bon :
- contexte clair
- action claire
- résultat clair

## Bon exemple 2
Quand je termine une réunion, je veux transformer immédiatement les décisions prises en tâches assignées, afin d'éviter que tout retombe dans l'oubli.

Pourquoi c'est bon :
- lié à un moment réel
- pain implicite visible
- résultat concret

## Bon exemple 3
Quand je compare les performances de plusieurs campagnes, je veux identifier vite ce qui fonctionne ou non, afin de couper les mauvaises décisions plus tôt.

## Bon exemple 4
Quand je dois documenter un process interne, je veux structurer l'information sans friction, afin que l'équipe puisse la retrouver et l'appliquer facilement.

## Bon exemple 5
Quand je lance un nouveau produit, je veux comprendre rapidement ce que les premiers utilisateurs ne comprennent pas, afin d'améliorer l'onboarding avant de scaler.

## Mauvais exemple 1
L'utilisateur veut un dashboard moderne.

Pourquoi c'est mauvais :
- ce n'est pas un job, c'est déjà une solution
- aucun contexte
- aucun résultat métier

## Mauvais exemple 2
Le client veut gagner du temps.

Pourquoi c'est mauvais :
- trop vague
- aucun moment précis
- aucun usage réel

## Mauvais exemple 3
L'utilisateur veut une IA.

Pourquoi c'est mauvais :
- l'IA n'est pas le job
- aucun besoin réel explicité

## Mauvais exemple 4
Le manager veut mieux gérer son équipe.

Pourquoi c'est mauvais :
- trop large
- pas de déclencheur
- pas d'outcome précis
