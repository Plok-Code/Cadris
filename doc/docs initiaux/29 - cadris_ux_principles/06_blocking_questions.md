# 06_blocking_questions

## Questions restantes

### Q-01 - Quels labels secondaires garder affiches en permanence, et lesquels rendre contextuels ?
- Pourquoi cela reste ouvert : la logique `label simple + terme expert secondaire` est retenue, mais le niveau exact de redondance peut varier selon les ecrans.
- Ce qu'il faut obtenir pour avancer : une regle de microcopie entre `toujours montrer les deux`, `simple d'abord puis aide contextuelle`, ou `double niveau seulement sur les termes critiques`.
- Hypothese temporaire : `double niveau seulement sur les termes critiques`.

### Q-02 - Quel niveau de reponse mobile est acceptable avant de basculer vers desktop ?
- Pourquoi cela reste ouvert : le scope `consultation + reponses` est retenu, mais il faut encore fixer la longueur et la complexite de reponse tolerees sur mobile.
- Ce qu'il faut obtenir pour avancer : une regle simple entre `reponses courtes seulement`, `reponses courtes + validation de reformulations`, ou `reponses longues ponctuelles`.
- Hypothese temporaire : `reponses courtes + validation de reformulations`.

### Q-03 - Quel wording exact retenir pour l'atterrissage post-intake ?
- Pourquoi cela reste ouvert : la structure `vue synthese puis CTA vers bloc actif` est retenue, mais le message d'accueil et le CTA principal peuvent encore etre affines.
- Ce qu'il faut obtenir pour avancer : la microcopie exacte du premier ecran de mission.
- Hypothese temporaire : `Votre mission est prete` + `Ouvrir le bloc actif`.

## Pourquoi ces points restent importants

- Q-01 conditionne la clarte des labels sans surcharger l'interface.
- Q-02 conditionne la limite ergonomique mobile.
- Q-03 conditionne la premiere impression juste apres l'intake.

## Priorite recommandee

1. regler la microcopie des labels secondaires ;
2. fixer la limite exacte des reponses mobiles ;
3. ecrire le wording post-intake.
