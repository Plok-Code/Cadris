# 04_iconography_style

## Style iconographique

Le style iconographique de Cadris doit etre :
- geometrique ;
- monoline ;
- structure ;
- lisible a petite taille ;
- suffisamment neutre pour vivre dans une interface dense.

## Principes d'icone

### Construction
- grille de base reguliere ;
- formes simples ;
- priorite aux verticales, horizontales et angles francs ;
- courbes presentes mais moderees ;
- sensation de cadre, de couche, de dossier, de liaison et de reperage.

### Trait
- trait principal fin a moyen ;
- epaisseur stable d'une icone a l'autre ;
- contrastes internes limites ;
- pas de micro-details ornementaux.

### Remplissage
- icones essentiellement en ligne ;
- usage ponctuel de surfaces remplies pour l'etat actif ou l'accent fort ;
- jamais de rendu 3D, glace, neon ou metal.

### Vocabulaire recommande
- dossier ;
- section ;
- pile de blocs ;
- lien ;
- repere ;
- annotation ;
- etat ;
- validation ;
- reserve ;
- transmission ;
- export ;
- progression ;
- architecture ;
- carte ;
- agent / orchestration via modules relies, jamais via personnages.

## Style d'illustration et d'imagerie

### Ce qu'il faut privilegier
- schemas de structure ;
- compositions de cartes, panneaux ou blocs ;
- fragments d'interfaces ;
- vues de dossier, annotations, surlignages, statuts ;
- representations abstraites de cooperation entre agents sous forme de couches ou modules relies ;
- textures tres legeres de trame, quadrillage ou papier mineral.

### Presence humaine
Hypothese de travail retenue :
**humain rare et discret**.

Concretement :
- pas de portraits hero ;
- pas d'equipe souriante type stock photo ;
- pas d'humanisation excessive des agents ;
- eventuellement des mains, un bureau, un geste de consultation, une presence contextuelle.

### Ce qu'il faut eviter
- robots, mascottes, avatars d'agents ;
- cerveaux, eclairs, etoiles, sparkles ;
- scenes futuristes abstraites ;
- isometries spectaculaires ;
- illustrations "friendly startup" trop souples ou trop enfantines ;
- trop d'humain qui ferait glisser la marque vers un service de conseil.

## Cohherence avec le systeme

L'iconographie et l'imagerie doivent renforcer les memes signaux que la palette et la typographie :
- clarte ;
- structure ;
- progression ;
- statut ;
- maitrise ;
- transmission.

Une bonne image Cadris doit expliquer ou orienter, pas seulement decorer.

## Usages recommandes

### Marque / landing
- illustrations de structure en hero ;
- apercus d'artefacts ;
- schemas de cooperation ou de progression ;
- details de composition plutot que grandes scenes narratives.

### Produit
- icones utilitaires ;
- reperes de navigation ;
- statuts ;
- marqueurs de certitude ;
- indications de blocage, validation et transmission.

### Dossier / export
- pictogrammes sobres ;
- index visuel ;
- reperes de section ;
- marqueurs de statut non agressifs.

## Limites d'usage

- pas plus d'un style d'icone principal ;
- pas de coexistence entre line icons rondes et pictos pleins agressifs ;
- pas d'illustration decorative dans les zones ou la lecture est critique ;
- pas de confusion entre icone de navigation et signal de statut ;
- ne jamais remplacer un texte important par une icone seule.

## Decision de travail

Le style iconographique et illustratif Cadris est :
**monoline, structurel, abstrait utile, centre sur les artefacts et les systemes**, avec une presence humaine faible et contextuelle seulement.
