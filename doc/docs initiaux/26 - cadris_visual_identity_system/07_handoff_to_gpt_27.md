# 07_handoff_to_gpt_27

## Resume executif

Le systeme d'identite visuelle de Cadris est maintenant suffisamment defini pour orienter une future direction logo.

Le cap retenu est clair :
- marque editoriale, structurelle, logicielle ;
- expression `expert accessible` ;
- palette claire et minerale ;
- accent principal petrol / bleu-vert maitrise ;
- systeme typographique documente sur base de litterature et d'ergonomie ;
- langage graphique fonde sur cadres, couches, reperes et connecteurs ;
- iconographie abstraite utile ;
- humain rare et discret.

## Systeme visuel retenu

- systeme clair-first et produit-compatible ;
- forte dominance de neutres mineraux ;
- lisibilite avant effet ;
- continuite entre acquisition, produit et export ;
- statuts visuellement distincts sans sur-dramatisation ;
- anti-references maintenues : neon IA, cabinet froid, cyber-ops, startup playful, wiki sans marque.

## Couleurs

- palette officielle : `Mineral petrole` ;
- base : `#FBFAF7`, `#E7E0D5`, `#CDC3B3`, `#2E3431` ;
- accent principal : `#3F7277` avec renfort `#234D52` ;
- statuts :
  - confirme / valide : `Sage`
  - hypothese / a confirmer : `Ochre`
  - inconnu / neutre : `Mist`
  - bloquant / reserve critique : `Brick`

Logique :
- la marque vit surtout dans les neutres + petrol ;
- les couleurs de statut restent fonctionnelles, jamais decoratives.

## Typographies

Base de travail retenue :
- `Public Sans + IBM Plex Mono`

Alternatives conservees a titre secondaire :
- `Inter + IBM Plex Mono`
- `IBM Plex Sans + IBM Plex Mono`

Piste accessibilite forte, a manier avec prudence :
- `Atkinson Hyperlegible Next + IBM Plex Mono`

Logique :
- une sans serif principale pour marque + produit ;
- une mono fonctionnelle en secondaire ;
- priorite aux criteres de lisibilite, de rythme, d'espacement, de longueur de ligne et de compatibilite ecran ;
- aucune validation sur le mythe d'une police "objectivement parfaite" ;
- `Public Sans` est retenue parce qu'elle sert le mieux la promesse de cadre clair, de confiance et d'accessibilite large.

## Principes graphiques

- grilles nettes ;
- blocs cadres ;
- filets et reperes ;
- couches et assemblages ;
- asymetrie disciplinee ;
- imagerie de systeme, d'artefact et de structure ;
- humain faible et contextuel.

## Style iconographique / illustratif

- monoline geometrique ;
- abstrait utile ;
- agents representes par orchestration, couches ou modules relies ;
- aucune mascotte, aucun robot, aucune iconographie magique.

## Points confirmes

- la marque ne doit pas commencer par un symbole logo complexe ;
- la force vient d'abord du wordmark, de la typographie et du systeme ;
- `expert accessible` est prefere a `premium discret` ;
- le systeme doit servir la marque et le produit des le debut ;
- les agents IA peuvent etre assumes conceptuellement, mais pas sur-joues graphiquement.
- la typo de travail retenue est `Public Sans + IBM Plex Mono`.

## Hypotheses de travail

- humain rare et discret ;
- dark mode pas prioritaire en V1 ;
- base typographique = sans robuste + mono fonctionnelle ;
- logo a venir plutot wordmark-first que symbole-first.

## Inconnus

- scope reel d'un dark mode ;
- inventaire complet du futur set d'icones ;
- niveau exact de motion dans le systeme final ;
- regle definitive d'usage entre logo monochrome et logo accent.

## Bloquants

- aucun bloquant strict pour transmettre vers GPT 27 ;
- seulement des sujets d'industrialisation a confirmer : dark mode, set d'icones, regles logo couleur.

## Niveau de fiabilite

- Niveau de fiabilite : Bon
- Raison : le systeme s'appuie sur une strategie visuelle deja convergente et sur des contraintes produit documentees ailleurs dans le projet. Les inconnus restants portent surtout sur l'industrialisation.

## Ce que le GPT 27 doit traduire en direction logo

1. Partir d'un **wordmark d'abord**, pas d'un pictogramme autonome.
2. Considerer que la tension du wordmark dependra de la piste typo choisie :
   - `Public Sans` pour la confiance et le cadre ;
   - `Inter` pour le produit et l'ecran ;
   - `IBM Plex Sans` pour le systeme humain-machine ;
   - `Atkinson` seulement comme piste accessibilite, pas comme choix par defaut de marque.
3. Chercher une logique de logo issue de :
   - la structure ;
   - le cadre ;
   - la mise en ordre ;
   - la liaison ;
   - les reperes.
4. Garder une geometrie sobre, precise, potentiellement issue de la construction des lettres plutot que d'un symbole exterieur plaque.
5. Eviter toute piste logo qui raconte :
   - l'IA magique ;
   - le robot ;
   - la vitesse ;
   - le cabinet ;
   - la documentation froide.
6. Considerer qu'un signe eventuel, s'il existe, doit rester secondaire au mot et pouvoir naitre d'une logique de cadre, de couche ou de repere.
7. Eviter un monogramme `C` trop generique ou trop corporate comme solution paresseuse.

## Ce que le GPT 27 doit traiter avec prudence

- toute piste trop symbolique trop tot ;
- toute recherche de singularite pure qui casserait la lisibilite ;
- toute serif statutaire ou trop luxe ;
- tout logo qui ferait croire a un cabinet ou a un app builder IA generique ;
- tout signe qui serait decoratif mais sans rapport avec le systeme visuel deja fixe.
