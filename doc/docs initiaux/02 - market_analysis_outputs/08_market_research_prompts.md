# Prompts de recherche marché

## 1. Prompt large d’analyse marché
Analyse le marché des outils qui aident des fondateurs, PM, designers ou développeurs à transformer une idée de SaaS en produit exécutable avec l’IA.

Je veux que tu cartographies :
- les catégories d’acteurs ;
- les jobs-to-be-done couverts ;
- les segments d’utilisateurs ;
- les promesses dominantes ;
- les tensions principales du marché ;
- les zones déjà saturées ;
- les zones encore fragmentées ;
- les signaux d’opportunité plausibles ;
- les risques de convergence rapide des leaders.

Je veux une réponse structurée en :
1. vue d’ensemble du marché
2. catégories d’acteurs
3. besoins bien couverts
4. besoins mal couverts
5. hypothèses d’opportunité
6. signaux favorables et défavorables
7. points qui demandent encore validation terrain

## 2. Prompt d’analyse concurrentielle
Analyse en profondeur les concurrents et alternatives de ce concept :

“Un système qui transforme une idée de SaaS en dossier d’exécution très poussé, cohérent, exportable, exploitable par des LLM de build, puis aide à contrôler la cohérence des sorties pendant l’exécution.”

Je veux que tu distingues :
- concurrents directs ;
- concurrents directs partiels ;
- concurrents indirects ;
- alternatives non logicielles ;
- bricolages multi-outils.

Pour chaque acteur identifié, précise :
- ce qu’il promet ;
- pour qui ;
- ce qu’il couvre réellement ;
- ses forces ;
- ses angles morts ;
- en quoi il menace ou non ce concept.

Termine par :
- une carte des overlaps ;
- les espaces de différenciation encore plausibles ;
- les risques d’être perçu comme “encore un builder IA”.

## 3. Prompt sur les alternatives non logicielles
Étudie comment des porteurs de projet peu expérimentés cadrent aujourd’hui un SaaS sérieux sans outil dédié de type “dossier canonique pour build IA”.

Je veux que tu identifies :
- les workflows manuels ;
- les bricolages les plus courants ;
- les prestataires ou profils humains qui compensent ce manque ;
- les coûts cachés ;
- les frustrations ;
- les raisons pour lesquelles ces alternatives restent attractives malgré leurs limites.

Je veux une réponse qui précise :
1. alternatives humaines
2. alternatives bricolées
3. non-consommation
4. coûts et limites
5. ce qu’un nouveau produit devrait battre pour être adopté

## 4. Prompt sur les signaux d’opportunité
Évalue s’il existe une vraie opportunité produit dans l’angle suivant :

“Réduire la dérive entre idée, cadrage, documentation et exécution dans les projets SaaS construits avec des LLM ou des agents de code.”

Je veux que tu recherches :
- des preuves que la vitesse actuelle du build IA crée des problèmes de cohérence, de qualité ou de transmission ;
- des signes que les outils existants traitent partiellement ce problème sans le résoudre complètement ;
- des signaux que certains utilisateurs seraient prêts à payer pour plus de structure, de justification et de contrôle ;
- les segments où cette douleur paraît la plus forte ;
- les contre-signaux qui affaiblissent cette opportunité.

Termine par :
- opportunité forte / moyenne / faible ;
- pourquoi ;
- ce qu’il faudrait encore prouver.

## 5. Prompt sur les objections, risques et limites du marché
Prends ce concept et attaque-le sérieusement :

“Un outil de cadrage profond et de dossier d’exécution pour SaaS buildés avec IA, avec boucle d’aval pour garder la cohérence.”

Je veux que tu listes les objections les plus fortes possibles :
- objections marché ;
- objections produit ;
- objections adoption ;
- objections monétisation ;
- objections concurrentielles ;
- objections de complexité ;
- objections liées à la vitesse d’évolution des leaders du marché.

Pour chaque objection :
- explique pourquoi elle est crédible ;
- estime sa gravité ;
- indique ce qui permettrait de la réfuter ou de la réduire.

Termine par :
- les 5 risques principaux ;
- les 3 risques existentiels ;
- les validations minimales à obtenir avant d’investir davantage.
