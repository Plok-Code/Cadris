# 06_blocking_questions

## Statut general

Aucun bloquant strict n'empeche de cloturer la chaine de cadrage.
Les questions ci-dessous bloquent surtout le passage d'une beta restreinte a un lancement plus ambitieux.

## Questions bloquantes restantes

### Q-01 - Quel scope exact fait foi au jour du lancement ?

Pourquoi cela bloque :
- sans scope final explicite, le lancement peut glisser vers PDF, share links ou flows secondaires et brouiller tous les signaux.

Ce qu'il faut obtenir pour avancer :
- une phrase de scope officielle ;
- une liste des surfaces hors scope ;
- un gate explicite pour toute extension.

### Q-02 - Quel canal fournit les 5 a 10 premiers utilisateurs qualifies ?

Pourquoi cela bloque :
- sans cohorte qualifiee, le lancement risque d'attirer surtout de la curiosite non exploitable.

Ce qu'il faut obtenir pour avancer :
- liste courte de canaux ;
- critere de qualification minimal ;
- mode d'invitation ou de selection.

### Q-03 - Quel niveau d'accompagnement humain est autorise pendant la vague 1 ?

Pourquoi cela bloque :
- sans regle simple, le feedback et les metriques peuvent etre artificiellement bons.

Ce qu'il faut obtenir pour avancer :
- cadre d'intervention ;
- mode de prise de notes ;
- regle de comptage des aides manuelles.

### Q-04 - Quel critere GO / NO GO fait passer de la vague 1 a la vague 2 ?

Pourquoi cela bloque :
- sans seuil de passage, le lancement progressif devient arbitraire.

Ce qu'il faut obtenir pour avancer :
- un petit ensemble de conditions minimales, par exemple :
  - completion suffisante ;
  - reprise saine ;
  - dossier juge utile ;
  - aide humaine non excessive.

### Q-05 - Quel niveau d'ouverture publique est vise apres la beta restreinte ?

Pourquoi cela bloque :
- la suite du lancement depend du niveau d'autonomie attendu, du support et du niveau de finition produit.

Ce qu'il faut obtenir pour avancer :
- decision entre :
  - beta elargie sur invitation ;
  - ouverture progressive avec file d'attente ;
  - lancement public plus large plus tard seulement.

## Decision de travail

Ces questions ne bloquent pas la fin du cadrage.
Elles bloquent surtout :
- l'operation de lancement ;
- le passage a l'echelle ;
- la lecture propre des apprentissages apres beta.
