# 02_non_goals

## Éléments explicitement exclus
- build autonome fort ;
- génération d’application de bout en bout ;
- gestion de projet générique ;
- documentation automatique “pour la doc” ;
- réparation d’un projet déjà cassé comme seule lecture du produit ;
- one-shot documentaire sans continuité minimale ;
- analytics secondaires ;
- permissions complexes ;
- rôles avancés ;
- collaboration riche multi-parties ;
- marketplace, templates avancés, éditeurs visuels complets ;
- gouvernance enterprise, sécurité profonde, conformité lourde ;
- support natif de projets ultra-spécifiques ou très réglementés ;
- multi-workflows complexes dès la V1.

## Raisons de ces exclusions
Ces éléments ne servent pas la preuve de valeur principale.
Ils déplacent le produit vers :
- un app builder ;
- un outil de doc ;
- un outil de gestion ;
- ou une couche enterprise trop lourde.

Ils cassent aussi la discipline MVP en ajoutant des fonctions “rassurantes” qui ralentissent l’apprentissage sans mieux prouver la valeur.

## Risques si on les réintègre trop tôt
- confusion de catégorie ;
- perte de lisibilité commerciale ;
- dilution de la promesse centrale ;
- hausse forte de complexité produit et technique ;
- MVP qui ressemble à une démo brouillonne ou à un mini-produit complet ;
- difficulté à tester la vraie boucle de valeur.
