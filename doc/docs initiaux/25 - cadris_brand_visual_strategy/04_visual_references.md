# 04_visual_references

## References utiles

### 1. Systeme editorial suisse contemporain
- Ce qu'il faut retenir :
  - grilles nettes ;
  - hierarchie forte ;
  - precision typographique ;
  - sobriete qui renforce la credibilite.
- Ce qu'il faut eviter de reproduire :
  - austerite excessive ;
  - rigidite sterile ;
  - froideur museale.

### 2. Plans d'architecture et legendes techniques
- Ce qu'il faut retenir :
  - logique de couches ;
  - annotations utiles ;
  - sentiment de structure visible ;
  - clarte des relations entre elements.
- Ce qu'il faut eviter de reproduire :
  - effet blueprint litteral ;
  - surcharge de traits ;
  - jargon visuel trop technique.

### 3. Interfaces produit sobres a haute precision, type Linear
- Ce qu'il faut retenir :
  - rigueur des composants ;
  - tension juste entre densite et clarte ;
  - mouvement discret ;
  - impression de produit serieux et affute.
- Ce qu'il faut eviter de reproduire :
  - aura trop sombre ;
  - imaginaire trop "elite product team" ;
  - sophistication qui exclut.

### 4. Clarte editoriale produit, type Notion
- Ce qu'il faut retenir :
  - evidence de l'artefact ;
  - lisibilite immediate ;
  - sensation d'outil exploitable ;
  - sobriete qui laisse la valeur parler.
- Ce qu'il faut eviter de reproduire :
  - neutralite trop banale ;
  - manque de tension de marque ;
  - lecture "simple outil de pages".

### 5. Signaletique architecturale et wayfinding haut de gamme
- Ce qu'il faut retenir :
  - notion de repere ;
  - guidance visuelle ;
  - assurance calme ;
  - intelligibilite dans l'espace.
- Ce qu'il faut eviter de reproduire :
  - severite institutionnelle ;
  - langage public ou administratif ;
  - abstraction trop froide.

## Anti-references

### 1. Le SaaS IA neon et cosmique
- A eviter :
  - halos lumineux ;
  - spheres ;
  - gradients violets / bleus trop attendus ;
  - promesse visuelle de magie autonome.
- Pourquoi :
  - cela contredit la posture anti-hype et anti-surpromesse de Cadris.

### 2. Le cabinet premium froid
- A eviter :
  - bleu marine + or ;
  - serif statutaires ;
  - mise en scene de conseil ou de direction generale ;
  - discours visuel trop institutionnel.
- Pourquoi :
  - Cadris doit rester un produit logiciel, pas un service cache.

### 3. Le hacker-brag ou cyber-ops
- A eviter :
  - noir integral ;
  - vert terminal ;
  - interfaces de controle agressives ;
  - fiertes de complexite technique.
- Pourquoi :
  - cela rapproche la marque d'un imaginaire d'operations ou de domination technique.

### 4. Le startup playful generique
- A eviter :
  - mascottes ;
  - illustrations trop mignonnes ;
  - couleurs confiserie ;
  - tonalite "outil sympa" sans tension.
- Pourquoi :
  - cela affaiblit la promesse de fiabilite et de tenue.

### 5. La documentation sans marque
- A eviter :
  - pages trop nues ;
  - interface purement utilitaire ;
  - absence de signe distinctif ;
  - impression de wiki ou de knowledge base.
- Pourquoi :
  - Cadris doit rester memorisable et identifiable, pas seulement fonctionnel.

## Synthese a retenir

La bonne reference pour Cadris n'est ni une marque IA spectaculaire, ni un cabinet, ni un wiki.

C'est un croisement entre :
- une culture editoriale exigeante ;
- une precision de produit logiciel ;
- une grammaire de structure inspiree de l'architecture ;
- une finition premium discrete.
