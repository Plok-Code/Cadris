# 02_brand_ui_alignment

## Alignement entre marque et interface

L'alignement global entre marque et interface est bon.

La marque promet :
- clarte ;
- cadre ;
- fiabilite ;
- accessibilite serieuse ;
- progression tenable.

L'interface raconte bien la meme chose avec :
- un shell sobre ;
- une hierarchie `mission-first` ;
- des statuts explicites ;
- des blocs documentaires ;
- une densite tenue ;
- une marque presente mais non theatrale.

## Ce qui est particulierement bien aligne

### Palette

`Mineral petrole` soutient correctement :
- la confiance ;
- la tenue editoriale ;
- la neutralite active ;
- la distinction entre structure et alerte.

### Typographie

`Public Sans + IBM Plex Mono` soutient bien :
- l'idee de cadre ;
- l'accessibilite large ;
- la lecture dense ;
- la presence systeme sans durete excessive.

### UI et UX

Le choix :
- synthese d'abord ;
- bloc actif ensuite ;
- questions fusionnees ;
- progression visible ;
- friction utile conservee

est tres coherent avec une marque qui veut rassurer sans faire spectacle.

### Logo

Le signe observe dans les assets :
- n'essaie pas de "faire IA" ;
- reste geometrique ;
- assume la structure ;
- tient la promesse de sobriete.

## Tensions entre expressivite et lisibilite

### 1. Le symbole est plus fort que le mot-symbole actuellement visible

Comme le pack actuel montre surtout des symboles, la marque risque d'etre percue plus par son signe que par le nom `CADRIS.AI`.

Impact possible :
- bonne memorisation du symbole ;
- mais faible stabilisation de la signature complete.

Correction minimale :
- finaliser une version canonique `symbole + texte`.

### 2. Le cadre peut devenir trop present dans l'app

La marque aime :
- les cadres ;
- les bords ;
- les reperes ;
- les panneaux.

L'UI aussi.

Risque :
- sur-cadrer l'interface ;
- rendre certaines zones plus institutionnelles que necessaire.

Correction minimale :
- reserver les gestes de marque les plus forts aux vues d'entree, au shell et a l'export ;
- garder les vues de travail plus sobres.

### 3. La mono peut vite durcir le ton

`IBM Plex Mono` est bien placee comme couche systeme.
Si elle deborde dans :
- des titres ;
- des contenus longs ;
- des labels majeurs ;

elle peut faire basculer la perception vers un produit plus technique que `expert accessible`.

Correction minimale :
- limiter la mono a la meta, aux statuts systeme et aux references.

### 4. La douceur des statuts peut affaiblir la promesse de securisation

La marque doit etre rassurante.
Si les etats critiques sont trop pales ou trop proches visuellement, le produit peut paraitre :
- soigne ;
- mais pas assez ferme sur ce qui bloque.

Correction minimale :
- renforcer la lisibilite des statuts sans perdre la retenue generale.

## Coherence percue

Perception attendue si le systeme est bien applique :
- la marque semble credible ;
- l'interface semble utile ;
- la densite reste tenable ;
- le produit parait maitrise sans etre froid.

Je n'observe pas de fracture entre :
- l'univers de marque ;
- la logique de travail ;
- le design system.

La coherence percue est donc :
**bonne a tres bonne**, avec un point d'attention sur la fermete visuelle des etats et sur la finalisation du lockup logo.

## Points a corriger

- figer un lockup `symbole + texte` officiel ;
- appliquer partout la doctrine unique de vocabulaire d'etat ;
- renforcer la lisibilite des tags de statut en petite taille ;
- eviter de dupliquer les cadres forts ou le symbole dans les ecrans deja denses.

## Decision d'alignement

L'ensemble marque + UI est :
**bien aligne, avec des tensions de calibration plus que des contradictions de fond**.
