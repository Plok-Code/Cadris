# 08_business_research_prompts.md

## 1. Prompt — business models du secteur
Tu es un analyste business model spécialisé en outils SaaS / IA pour builders.

Contexte :
Le produit aide des solo founders, indie builders ou micro-équipes à transformer un proto ou une petite app construite avec IA en projet SaaS plus fiable, grâce à une source de vérité, un dossier canonique et une meilleure continuité de décision.

Mission :
analyse les business models les plus plausibles pour ce type de produit.

Je veux que tu compares :
- abonnement simple ;
- abonnement ancré sur projet actif ;
- paiement par projet ;
- hybride projet + continuité ;
- setup + abonnement ;
- usage-based secondaire.

Pour chaque modèle :
- qui paie ;
- pour quoi il paie ;
- à quel moment il paie ;
- pourquoi ce modèle est crédible ou fragile ;
- ce qu’il suppose sur la fréquence d’usage ;
- ce qu’il implique pour la rétention ;
- ce qu’il implique pour le support et la marge.

Termine par :
- 1 modèle recommandé ;
- 1 modèle alternatif ;
- 3 risques de mauvais choix ;
- un registre Confirmé / Hypothèse / Inconnu / Bloquant.

## 2. Prompt — structures de pricing comparables
Tu es un chercheur pricing.

Sujet :
Je veux comprendre comment des produits comparables structurent leur pricing quand ils vendent :
- de la continuité de travail ;
- de la mémoire projet ;
- de la réduction de dérive ;
- ou un livrable de structuration exploitable par d’autres outils.

Mission :
repère des structures de pricing comparables sans copier bêtement les prix.

Je veux que tu identifies :
- l’unité de valeur visible ;
- l’unité de facturation ;
- la présence ou non de paliers ;
- la place éventuelle des crédits ;
- les choix qui réduisent la friction psychologique ;
- les formulations qui rendent l’offre concrète.

Termine par :
- patterns récurrents ;
- patterns à éviter ;
- ce qui semble transposable ;
- ce qui ne l’est pas.

## 3. Prompt — unités de valeur et unités de facturation
Tu es un spécialiste du pricing produit.

Contexte :
Le produit peut créer de la valeur via :
- un dossier canonique ;
- une source de vérité ;
- une mémoire continue ;
- un handoff ;
- une réduction du risque de rebuild.

Mission :
détermine quelles unités de valeur et quelles unités de facturation sont les plus cohérentes.

Compare :
- projet actif ;
- dossier / export ;
- workspace ;
- collaborateur ;
- profondeur ;
- run / analyse ;
- crédits ;
- hybride.

Pour chaque unité :
- lien avec la valeur perçue ;
- lisibilité ;
- risque de stress ou d’effet compteur ;
- compatibilité avec solo founder ;
- compatibilité avec micro-équipe ;
- impact sur la rétention ;
- fragilité opérationnelle.

Termine par :
- unité principale recommandée ;
- unité secondaire possible ;
- unités à éviter.

## 4. Prompt — signaux de willingness to pay
Tu es un chercheur en validation de willingness to pay.

Sujet :
Un produit aide des builders IA déjà un peu avancés à éviter la dérive, garder une source de vérité et transmettre un projet SaaS sans repartir de zéro.

Mission :
identifie les signaux montrant si ce marché paiera vraiment, et pour quoi exactement.

Je veux que tu me donnes :
- douleurs qui déclenchent un achat ;
- comportements qui indiquent déjà un coût subi ;
- objections économiques probables ;
- formulations d’offre les plus achetables ;
- formulations faibles ;
- questions d’entretien à poser ;
- signaux d’achat faibles vs forts.

Termine par :
- ce qui semble vendable ;
- ce qui semble utile mais peu achetable ;
- les hypothèses à tester immédiatement.

## 5. Prompt — risques économiques ou objections au pricing
Tu es un expert en objections de pricing.

Contexte :
Le produit n’est ni un simple builder IA, ni une simple base documentaire. Il vend une réduction du chaos, de la dérive et du risque de rebuild.

Mission :
identifie les objections économiques et de packaging les plus probables.

Je veux que tu analyses :
- “je peux déjà faire ça avec Claude / Cursor / ChatGPT / Notion” ;
- “c’est trop de doc” ;
- “ça me ralentit” ;
- “je paie déjà trop d’outils IA” ;
- “je ne comprends pas exactement ce que j’achète” ;
- “je préfère un humain pour ça”.

Pour chaque objection :
- ce qu’elle révèle ;
- sa gravité ;
- comment la tester ;
- comment la réduire par le produit ;
- comment la réduire par le wording ;
- comment elle influence le pricing.

Termine par :
- les objections les plus dangereuses ;
- les objections gérables ;
- les tests recommandés en priorité.
