# 06_blocking_questions.md

## 1. Quelle phrase décrit le mieux la douleur d’achat principale ?
### Pourquoi cela bloque
Le produit peut être acheté pour :
- éviter un rebuild ;
- arrêter la dérive ;
- obtenir une vraie source de vérité ;
- professionnaliser un proto ;
- transmettre sans réexpliquer tout le contexte.

Sans hiérarchie claire, le message restera flou.

### Ce qu’il faut obtenir
- 1 phrase principale ;
- 1 à 2 douleurs secondaires maximum ;
- le vocabulaire exact de l’utilisateur.

## 2. Quel wedge SaaS exact sert-on en premier ?
### Pourquoi cela bloque
“Vrai SaaS” reste trop large.

### Ce qu’il faut obtenir
- 3 cas explicitement inclus ;
- 3 cas explicitement exclus ;
- un niveau de complexité acceptable ;
- un angle simple de type micro-SaaS B2B mono-workflow ou équivalent.

## 3. Le produit vend-il surtout un dossier exportable ou une continuité vivante ?
### Pourquoi cela bloque
Le business model, le pricing et la roadmap changent selon que l’on vend :
- un livrable concret ;
- une mémoire continue ;
- un système hybride.

### Ce qu’il faut obtenir
- la forme de valeur la plus vite comprise ;
- la forme de valeur la plus vite achetée ;
- la forme de valeur la plus retenue.

## 4. Le produit intervient-il avant le code, juste après le proto, ou aussi pendant l’exécution ?
### Pourquoi cela bloque
Le moment d’entrée change :
- la promesse ;
- l’onboarding ;
- la structure de revenu ;
- la charge support.

### Ce qu’il faut obtenir
- un point d’entrée principal ;
- un point d’entrée secondaire ;
- ce que le produit ne promet pas encore.

## 5. Le sous-segment post-Lovable / post-builder bloqué est-il un vrai segment acheteur ?
### Pourquoi cela bloque
Il est séduisant intellectuellement, mais peut n’être qu’un angle de wording.

### Ce qu’il faut obtenir
- entretiens directs ;
- motifs de frustration récurrents ;
- capacité réelle à payer ;
- signal qu’ils achètent autre chose qu’une migration improvisée.

## 6. Quel niveau de profondeur documentaire est suffisamment puissant sans devenir lourd ?
### Pourquoi cela bloque
Trop peu de profondeur affaiblit la valeur.
Trop de profondeur ralentit l’achat et l’usage.

### Ce qu’il faut obtenir
- le minimum crédible ;
- le niveau premium ;
- le format préféré : dossier, mémoire, export, checklist de décisions, autre.

## Priorité de résolution
1. douleur principale ;
2. wedge exact ;
3. objet vendu exact ;
4. moment d’entrée produit ;
5. profondeur documentaire ;
6. validation du sous-segment post-Lovable.
