# 02_logo_constraints

## Contraintes de lisibilite

Le logo doit rester lisible dans quatre contextes prioritaires :
- header web ;
- favicon ;
- avatar / profil social ;
- document exporte.

Cela impose :
- une construction simple ;
- peu de details internes ;
- peu de contrastes decoratifs ;
- une silhouette claire ;
- un logotype qui ne depend pas d'effets ou de textures.

## Contraintes de reduction

### Petite taille
- le symbole doit rester reconnaissable a `16x16`, `24x24` et `32x32` ;
- aucun contre-forme critique ne doit disparaitre en dessous de `24px` ;
- les espaces interieurs doivent rester ouverts ;
- les traits trop fins ou doubles filets sont a eviter.

### Header
- le logotype complet doit bien tenir en bandeau horizontal ;
- il ne doit pas demander trop de largeur ;
- le symbole ne doit pas voler la vedette au nom.

### Documents et exports
- le logo doit exister en version monochrome claire et foncee ;
- il doit bien tenir en en-tete PDF ou Markdown converti ;
- il ne doit pas paraitre trop technologique ou trop ludique dans un contexte documentaire.

## Contraintes monochrome

Le logo doit fonctionner en :
- noir / encre sur fond clair ;
- blanc sur fond fonce ;
- une seule couleur petrol ;
- une seule couleur graphite.

Le sens du signe ne doit pas dependre :
- d'un degrade ;
- d'une superposition translucide ;
- d'un effet de lumiere ;
- d'une difference de couleur trop subtile.

## Contraintes de systeme visuel

Le logo doit rester coherent avec :
- palette minerale claire ;
- accent petrol maitrise ;
- typographie sans serif robuste ;
- langage graphique a base de cadres, couches, reperes et connecteurs ;
- iconographie monoline structurelle.

Il ne doit pas introduire un autre univers formel.

## Contraintes de perception

Le logo ne doit pas faire croire a :
- un app builder IA generique ;
- un generateur de documentation ;
- un cabinet de conseil ;
- un outil cyber / securite ;
- un produit enterprise froid.

Il doit plutot faire sentir :
- clarte ;
- structure ;
- fiabilite ;
- continuite ;
- projet tenable.

## Points de vigilance par famille

### Cadre / repere
- ne pas tomber dans le carre vide ;
- ne pas produire un signe type "camera / scan / crop tool" ;
- ne pas sur-corporatiser.

### Couche / assemblage
- ne pas empiler trop d'elements ;
- ne pas dessiner un schema produit miniature ;
- ne pas perdre la silhouette globale.

### Monogramme structure
- ne pas faire un `C` banal ;
- ne pas deformater la lettre jusqu'a l'illisibilite ;
- ne pas faire un monogramme startup paresseux.

## Tailles minimales recommandees

### Logotype + symbole
- digital : minimum `120px` de large recommande ;
- print : minimum `25mm` de large recommande.

### Symbole seul
- digital : minimum `16px` avec version simplifiee si necessaire ;
- print : minimum `6mm`.

## Test minimal obligatoire pour chaque piste

Avant validation, chaque piste doit etre testee en :
- noir sur blanc ;
- blanc sur petrol fonce ;
- symbole seul en `16px` ;
- logotype en header ;
- avatar carre ;
- en-tete de document.

## Decision de travail

La contrainte dominante de Cadris est la suivante :
**un vrai logo oui, mais avec une discipline severe de reduction et de sobriete**.
