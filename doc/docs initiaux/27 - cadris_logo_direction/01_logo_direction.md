# 01_logo_direction

## Role du logo

Le logo de Cadris n'a pas pour role de raconter tout le produit.
Son role est plus precis :
- rendre la marque identifiable tres vite ;
- installer une sensation de structure, de clarte et de maitrise ;
- fonctionner sur des usages de petite taille sans perdre son serieux ;
- accompagner un systeme visuel deja fort, pas le remplacer ;
- aider la memorisation de `CADRIS.AI` sans glisser vers l'imaginaire IA generique.

Le logo doit donc rassurer et orienter plus qu'impressionner.

## Type recommande

Type retenu :
**combination mark sobre**

Concretement :
- un logotype principal `CADRIS.AI` ou `CADRIS` selon les contextes ;
- un symbole autonome simple ;
- une relation stable entre les deux, sans dependance a un symbole trop bavard.

Ce choix est retenu parce que l'utilisateur veut un vrai logo, pas un mot-symbole seul, et parce que Cadris aura besoin d'un signe reduit pour :
- favicon ;
- avatar social ;
- app icon ;
- repere de header ;
- signature de document.

## Niveau de symbolique

Niveau recommande :
**modere-faible**

Le symbole doit evoquer :
- cadre ;
- structure ;
- couche ;
- repere ;
- mise en ordre ;
- liaison ;

Mais il ne doit pas essayer de raconter :
- l'ensemble du systeme multi-agent ;
- la documentation ;
- l'architecture logicielle ;
- la securite ;
- l'intelligence artificielle au sens spectaculaire.

## Direction generale retenue

La direction logo de Cadris est :
**un logo de structure claire, compose d'un logotype stable et d'un symbole reductible, construit sur une logique de cadre, d'assemblage ou de monogramme structure**.

Le ton doit rester :
- expert accessible ;
- serieux mais non institutionnel ;
- memorisable sans theatrie ;
- compatible avec un systeme editorial et produit.

## Trois familles de logo a produire et tester

L'objectif n'est pas de choisir une seule famille trop tot.
Il faut produire les 3 familles, avec plusieurs sous-idees chacune, puis comparer.

### Famille A - Cadre / repere

#### Idee centrale
Un symbole qui evoque le cadre, la delimitation, le point d'appui, la mise au net.

#### Pourquoi cette famille est plausible
- elle est directement alignee avec le message `donne un cadre` ;
- elle est facile a lire a petite taille ;
- elle tient bien en favicon ;
- elle reste anti-hype.

#### Sous-idees a explorer

##### A1 - Cadre ouvert
- un cadre incomplet ou ouvert sur un cote ;
- suggere qu'on structure sans enfermer ;
- bon fit avec "le builder reste auteur".

##### A2 - Double repere d'angle
- deux ou quatre coins de cadrage tres simples ;
- evoque la mise au point ou la delimitation d'un perimetre ;
- forte reduction possible.

##### A3 - Cadre + noyau
- un cadre simple avec un point, un bloc ou un petit module au centre ;
- suggere ce qui fait foi a l'interieur d'une structure claire ;
- attention a ne pas ressembler a une cible ou a une camera.

##### A4 - Repere de marge
- symbole qui rappelle un repere editorial ou une marque de section ;
- plus original, plus "architecture editoriale calme" ;
- a tester si l'on veut quelque chose de moins banal qu'un carre.

#### Risques de cette famille
- trop corporate ;
- trop "outil de scan" ;
- trop generique si reduit a un carre vide.

### Famille B - Couche / assemblage

#### Idee centrale
Un symbole compose de 2 ou 3 elements relies, emboites ou superposes, qui evoque coherence, systeme et transmission.

#### Pourquoi cette famille est plausible
- elle parle mieux du produit que la famille A ;
- elle peut mieux assumer la dimension multi-agent sans la surjouer ;
- elle peut produire un signe plus distinctif.

#### Sous-idees a explorer

##### B1 - Cartes empilees
- deux ou trois plaques / cartes legerement decalees ;
- evoque dossier, couches, arbitrages, structure ;
- bon lien avec les artefacts et documents.

##### B2 - Modules relies
- petits blocs relies par une liaison simple ;
- evoque cooperation et orchestration ;
- a garder tres minimal pour ne pas devenir technique.

##### B3 - Superposition verrouillee
- deux couches qui s'alignent partiellement pour former un ensemble stable ;
- suggere la coherence gagnee ;
- bonne piste pour montrer "ca tient".

##### B4 - Pile + repere
- base en couches avec un coin ou repere de cadrage ;
- permet de croiser structure produit et promesse "cadre clair".

#### Risques de cette famille
- trop complexe en reduction ;
- trop proche d'un schema produit ;
- trop "software architecture" si les liaisons sont trop visibles.

### Famille C - Monogramme structure

#### Idee centrale
Un symbole derive du `C` ou d'une combinaison de lettres, travaille comme forme de structure et non comme initiale decorative.

#### Pourquoi cette famille est plausible
- elle ancre fortement la memorisation de la marque ;
- elle evite le symbole generique de categorie ;
- elle peut bien fonctionner pour favicon et avatar.

#### Sous-idees a explorer

##### C1 - C cadre
- un `C` construit comme une forme de cadre ou de contenant ouvert ;
- simple et tres reductible ;
- demande une execution tres nette pour ne pas sembler trop generique.

##### C2 - C + repere
- `C` accompagne d'une coupe, d'un angle ou d'un point d'appui ;
- meilleur potentiel de signature ;
- bon compromis entre sobriete et memorisation.

##### C3 - CA structure
- monogramme `C` + `A` fusionne ou emboite ;
- plus distinctif ;
- risque plus fort de perdre la lisibilite en petit.

##### C4 - C en couches
- un `C` forme par 2 ou 3 strates ;
- bon lien avec la notion de structure et d'assemblage ;
- attention a ne pas produire un symbole trop "cloud / data / infra".

#### Risques de cette famille
- monogramme startup trop banal ;
- `C` trop corporate ;
- signe trop formaliste si la lettre n'est plus lisible.

## Recommandation d'ordre de test

Pour la production exploratoire, ordre recommande :
1. **Famille C - Monogramme structure**
2. **Famille A - Cadre / repere**
3. **Famille B - Couche / assemblage**

Pourquoi :
- C peut produire le meilleur signe de memorisation ;
- A donne la piste la plus simple et la plus robuste ;
- B est la plus prometteuse produit, mais aussi la plus risquee en reduction.

## Criteres de comparaison entre les pistes

Chaque piste devra etre jugee sur :
- memorisation ;
- lisibilite a petite taille ;
- coherence avec `CADRIS.AI` ;
- sensation de clarte et de tenue ;
- absence de confusion avec app builder, doc tool ou cabinet ;
- tenue en monochrome ;
- compatibilite favicon / app icon / header.

## Decision de travail

La direction logo de Cadris est donc :
- **un vrai logo avec symbole + logotype** ;
- **trois familles produites pour test** ;
- **plusieurs sous-idees par famille** ;
- avec une recommandation initiale orientee vers le **monogramme structure**, sans exclure les deux autres familles.
