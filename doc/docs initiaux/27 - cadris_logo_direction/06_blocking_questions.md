# 06_blocking_questions

## Questions bloquantes restantes

### BQ-01 - Quelle piste typographique principale accompagne le logo ?
- Pourquoi cela bloque : le dessin du logotype et la relation symbole / mot changent selon la famille typo retenue.
- Ce qu'il faut obtenir pour avancer : arbitrage entre les pistes typographiques de l'etape 26.
- Hypothese temporaire : `Public Sans + IBM Plex Mono`.

### BQ-02 - Quelle famille de symbole resiste vraiment a la reduction ?
- Pourquoi cela bloque : sur le papier, les trois familles sont plausibles ; seule l'execution montrera laquelle tient en `16px`, `24px`, `32px`.
- Ce qu'il faut obtenir pour avancer : produire les 3 familles et leurs sous-idees minimales, puis tester header, avatar et favicon.
- Hypothese temporaire : `monogramme structure` tient le mieux.

### BQ-03 - L'app icon est-elle un besoin reel de lancement ou un usage secondaire ?
- Pourquoi cela bloque : si l'app icon devient critique, il faudra pousser le symbole autonome plus loin des maintenant.
- Ce qu'il faut obtenir pour avancer : une hierarchie claire entre `favicon`, `avatar social`, `header`, `documents`, `app icon`.
- Hypothese temporaire : `favicon + header + avatar` d'abord, `app icon` ensuite.

## Pourquoi ces points bloquent

- BQ-01 change le ton du logotype.
- BQ-02 change la famille de symbole retenue.
- BQ-03 change le niveau de simplification du signe.

## Priorite recommandee

1. test graphique des trois familles ;
2. arbitrage typo principal ;
3. clarification de l'app icon.

- Bloquant pour definir une direction logo exploratoire : non.
