# 04_favicon_guidelines

## Principe favicon

Le favicon Cadris ne doit pas etre le logotype reduit.
Il doit etre :
- le symbole seul ;
- simplifie si necessaire ;
- parfaitement lisible en tres petite taille.

## Priorite de reduction

Ordre de priorite :
1. silhouette globale ;
2. contre-forme principale ;
3. memorisation ;
4. coherence avec le logo principal.

Tout ce qui est secondaire doit pouvoir disparaitre.

## Regles de simplification

- supprimer les details internes non essentiels ;
- epaissir legerement les traits si besoin ;
- augmenter les ouvertures ;
- preferer un signe plein ou semi-ouvert a un signe trop filete ;
- eviter tout double contour.

## Principe app icon

Si une app icon est necessaire plus tard, elle doit reprendre :
- le symbole seul ;
- dans un contenant simple si besoin ;
- avec fond uni ou tres sobre ;
- sans effet 3D ni mise en scene.

L'app icon ne doit pas inventer un nouvel univers.

## Recommandations par famille

### Cadre / repere
- tres favorable au favicon ;
- attention a ne pas finir en simple coin de recadrage trop banal.

### Couche / assemblage
- plus risque en reduction ;
- necessite probablement une version micro-simplifiee dediee.

### Monogramme structure
- souvent la meilleure famille pour favicon ;
- a condition que la lettre ou la forme reste distincte a 16px.

## Points critiques de reduction

### A 32px
- le signe doit deja etre stable ;
- les details secondaires peuvent encore tenir.

### A 24px
- seule la logique centrale doit rester ;
- les doubles lectures complexes doivent deja disparaitre.

### A 16px
- une seule idee visuelle doit survivre ;
- si le signe devient ambigu, il faut une version plus simple.

## Couleur favicon

Version recommandee :
- symbole petrol fonce sur fond clair ;
- ou symbole clair sur fond petrol fonce.

Version de secours :
- noir / blanc strict.

Eviter :
- gradients ;
- statuts ;
- multicolore ;
- effets de profondeur.

## Test obligatoire

Pour chaque famille et sous-idee :
- tester en `16x16`, `24x24`, `32x32` ;
- tester sur navigateur clair et sombre ;
- tester dans un onglet charge visuellement ;
- tester en avatar cercle et carre.

## Hypothese de travail

La famille la plus prometteuse pour favicon est actuellement :
**Monogramme structure**

Pourquoi :
- meilleure memorisation ;
- meilleure reduction ;
- meilleure coherence avec l'idee d'un vrai logo a signe.

## Decision de travail

Le favicon Cadris doit etre pense comme un micro-signe autonome, pas comme une miniature du logotype.
