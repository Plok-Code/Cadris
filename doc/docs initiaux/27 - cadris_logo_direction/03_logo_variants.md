# 03_logo_variants

## Variantes necessaires

Le systeme logo de Cadris doit prevoir des variantes des le depart.
Pas pour multiplier les styles, mais pour couvrir les usages reels.

## Variante 1 - Signature principale horizontale

### Composition
- symbole a gauche ;
- logotype a droite ;
- usage standard de marque.

### Usage
- header web ;
- landing ;
- presentations ;
- documents ;
- signatures de produit.

### Limites
- pas adaptee aux tres petites tailles ;
- ne pas utiliser en favicon ;
- ne pas serrer exagerement l'espace entre symbole et mot.

## Variante 2 - Logotype seul

### Composition
- `CADRIS.AI` ou `CADRIS` sans symbole.

### Usage
- contextes editoriaux ;
- grands titres de page ;
- situations ou le nom suffit ;
- supports ou l'encombrement doit rester minimal.

### Limites
- ne remplace pas le symbole pour les petits formats ;
- a ne pas utiliser comme cache-misere si le symbole est encore faible.

## Variante 3 - Symbole seul

### Composition
- signe autonome issu d'une des familles retenues.

### Usage
- favicon ;
- avatar social ;
- app icon ;
- repere de navigation ;
- watermark discret.

### Limites
- doit rester tres simple ;
- ne doit pas etre utilise seul dans des contextes de faible notoriete quand la marque n'est pas encore connue ;
- ne doit pas devenir plus "important" que le mot.

## Variante 4 - Signature compacte empilee

### Composition
- symbole au-dessus ou a gauche d'un logotype plus compact ;
- uniquement si la geometrie finale le justifie.

### Usage
- cartes de presentation ;
- slides ;
- zones plus verticales ;
- certains profils sociaux.

### Limites
- optionnelle ;
- a ne pas creer si elle semble artificielle ;
- ne doit pas produire une nouvelle marque parallele.

## Variante 5 - Monochrome

### Composition
- version noire / encre ;
- version blanche ;
- version petrol unie.

### Usage
- documents ;
- impression ;
- signatures sobres ;
- surfaces monochromes.

### Limites
- toute version qui perd son sens sans couleur doit etre rejetee.

## Variantes par famille de concept

Comme tu veux faire tester les trois familles, chaque famille doit sortir au minimum avec :
- une signature principale ;
- un symbole seul ;
- une version monochrome ;
- un comportement en favicon.

## Sous-idees a produire par famille

### Famille A - Cadre / repere
- A1 Cadre ouvert
- A2 Double repere d'angle
- A3 Cadre + noyau
- A4 Repere de marge

### Famille B - Couche / assemblage
- B1 Cartes empilees
- B2 Modules relies
- B3 Superposition verrouillee
- B4 Pile + repere

### Famille C - Monogramme structure
- C1 C cadre
- C2 C + repere
- C3 CA structure
- C4 C en couches

## Grille de test recommandee

Pour comparer les sous-pistes, il faut produire pour chacune :
- la version principale ;
- la version noir / blanc ;
- la version 32px ;
- la version 16px ;
- la version avatar carre ;
- la version header.

## Coherence avec le systeme visuel

Toutes les variantes doivent respecter :
- la meme logique de formes ;
- la meme tension typographique ;
- la meme retenue colorielle ;
- la meme promesse de clarte et de structure.

Il ne doit jamais y avoir :
- une variante fun ;
- une variante premium dramatisee ;
- une variante "IA" plus flashy ;
- une variante app icon qui semblerait venir d'une autre marque.

## Decision de travail

Le bon systeme logo pour Cadris n'est pas un seul fichier.
C'est un ensemble restreint de variantes strictement reliees, au service des usages critiques.
