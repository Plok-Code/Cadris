# 02_tone_of_voice

## Ton recommandé
Le ton recommandé pour Cadris est :

**direct, clair, concret, posé et crédible.**

La marque doit parler comme quelqu’un qui :
- comprend le chaos des projets construits avec l’IA ;
- sait remettre de l’ordre ;
- ne surjoue ni la performance ni la disruption ;
- aide à décider et à avancer.

## Niveau de formalité
**Moyen**

Recommandation actuelle :
- registre professionnel ;
- vocabulaire accessible ;
- phrases nettes ;
- préférence pour le vouvoiement en communication de marque.

Pourquoi :
- la cible est autonome et sérieuse ;
- un ton trop relâché ferait perdre de la crédibilité ;
- un ton trop institutionnel ferait glisser la marque vers le cabinet, le conseil ou l’outil corporate froid.

## Niveau de technicité
**Moyen**

Cadris doit pouvoir parler de structure, de cohérence, de transmission, de build, de workflow, de décisions, de dossier canonique.

En revanche, le ton ne doit pas :
- saturer de jargon technique ;
- chercher à impressionner ;
- donner l’impression que la valeur dépend d’un niveau d’expertise inaccessible.

Principe :
**assez technique pour être crédible, assez simple pour rester actionnable.**

## Niveau de proximité
**Moyen**

La marque doit rester :
- compréhensive ;
- utile ;
- non jugeante.

Elle peut reconnaître la fatigue, le flou, la recontextualisation pénible, la peur de casser l’existant.
Mais elle ne doit pas :
- surjouer la complicité ;
- parler comme un créateur de contenu ;
- basculer dans le coaching émotionnel.

## Règles de ton à respecter

### 1. Parler en bénéfices concrets
Toujours rattacher le discours à des usages tangibles :
- reprendre ;
- modifier ;
- transmettre ;
- retrouver ce qui fait foi ;
- éviter le rebuild aveugle.

### 2. Préférer les formulations sobres
Exemples de registre juste :
- “reprendre sans repartir de zéro”
- “garder le cap pendant le build”
- “transformer le flou en base exploitable”
- “éviter la dérive silencieuse”

### 3. Éviter la hype
À éviter :
- “révolutionner”
- “booster x10”
- “magique”
- “autonome de bout en bout”
- “from idea to app instantly”

### 4. Ne jamais vendre la doc pour la doc
Le discours doit toujours relier la forme documentaire à la valeur réelle :
- contrôle ;
- continuité ;
- fiabilité ;
- transmission.

### 5. Assumer les limites
Le ton gagne en crédibilité quand Cadris dit clairement ce qu’il ne fait pas :
- ne remplace pas les outils de build ;
- ne code pas tout à votre place ;
- ne prétend pas supprimer toute complexité.

### 6. Donner une impression d’ordre
Phrases plutôt courtes.
Enchaînements logiques.
Hiérarchie claire.
Pas de dispersion rhétorique.

### 7. En pricing, raconter la valeur et non le compute
Quand la marque parle de prix ou d’offre, elle doit raconter :
- la remise au carré ;
- la source de vérité ;
- la continuité du projet ;
- la réduction du risque de rebuild ou de perte de contexte.

À éviter comme façade principale :
- un langage de “runs” ;
- un compteur visible ;
- une narration trop centrée sur les crédits ;
- une justification du prix par la seule consommation technique.

Principe :
**Cadris vend d’abord une base projet plus tenable, pas des unités de calcul.**

## Formulations à favoriser
- “ce qui fait foi”
- “source de vérité”
- “continuité du projet”
- “mémoire de décision”
- “base exploitable”
- “projet tenable”
- “sans se reperdre”
- “sans rebuild aveugle”
- “garder la cohérence”
- “rendre le projet transmissible”

## Formulations à éviter
- “on build pour vous”
- “documentation automatisée”
- “copilote total”
- “plus besoin de réfléchir”
- “ship plus vite” comme promesse principale
- “tout gérer de A à Z”
- “productivité infinie”
- “stack parfaite automatiquement”

## Exemple de ton juste
“Cadris transforme un projet IA fragile en base claire et exploitable, pour reprendre, modifier ou transmettre sans se reperdre.”

## Exemple de ton trop faible
“Cadris génère une documentation intelligente pour vos projets.”

## Exemple de ton trop agressif
“Arrêtez de bricoler. Cadris remplace enfin vos workflows obsolètes.”

## Résumé opératoire
Le ton Cadris doit faire sentir :
- plus de clarté ;
- plus de contrôle ;
- moins de théâtre ;
- moins de flou ;
- plus de tenue dans le temps.
