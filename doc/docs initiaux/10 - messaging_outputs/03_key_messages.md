# 03_key_messages

## Ligne de message recommandée
### Hook principal
**Sans cadre clair, un projet vibe codé avance mais ne tient pas.**

### Message maître
**Cadris donne à un projet construit avec l’IA un cadre clair pour avancer sans dériver.**

### Sous-message
**Le problème n’est pas seulement d’aller vite. Le problème est de garder le projet cohérent quand la complexité commence vraiment.**

## Messages par cible

### Cible 1 — Vibe codeurs / builders IA déjà actifs
#### Ce qu’ils vivent
Le projet sort vite, mais plus il avance, plus il devient fragile à modifier et difficile à comprendre.

#### Message recommandé
Cadris donne au projet le cadre clair qui évite la dérive silencieuse pendant le build.

#### Formulations recommandées
- Votre projet avance. La question est : est-ce qu’il tient ?
- Ce qui manque n’est pas plus de vitesse. C’est ce qui fait foi.
- Cadris pose la base qui permet à un projet vibe codé de rester cohérent.

#### Angle prioritaire
Anti-dérive, anti-chaos silencieux, tenue du projet.

### Cible 2 — Builders SaaS non définis comme vibe codeurs
#### Ce qu’ils vivent
Ils utilisent l’IA de manière pragmatique mais cherchent surtout à faire tenir un produit dans le temps.

#### Message recommandé
Cadris donne à un projet construit avec l’IA une base claire pour rester modifiable, lisible et transmissible.

#### Formulations recommandées
- Quand un projet avance sans cadre clair, il devient difficile à faire évoluer.
- Cadris aide à garder la cohérence du projet à mesure qu’il grandit.
- La vraie valeur n’est pas de générer plus. C’est de conserver une base exploitable.

#### Angle prioritaire
Cohérence, continuité, transmissibilité.

### Cible 3 — Fondateur produit, PM, designer ou operator très hands-on
#### Ce qu’ils vivent
Ils savent ce qu’ils veulent construire, mais n’ont pas toujours la structure nécessaire pour tenir le projet quand il devient plus réel.

#### Message recommandé
Cadris clarifie ce qui fait foi pour que le projet reste compréhensible, modifiable et transmissible.

#### Formulations recommandées
- Cadris rend visibles les décisions qui doivent rester stables.
- Quand tout dépend du contexte implicite, le projet devient fragile.
- Cadris transforme le flou en base exploitable.

#### Angle prioritaire
Lisibilité, décisions, handoff.

## Angles de communication prioritaires
### 1. La dérive
Le projet peut sembler avancer alors qu’il perd sa cohérence.

### 2. Le cadre clair
Ce qui manque n’est pas forcément un outil de build supplémentaire, mais une base claire.

### 3. Ce qui fait foi
Le produit doit reposer sur une source de vérité et une mémoire de décision.

### 4. La tenue dans le temps
Un projet utile n’est pas seulement lancé ; il doit rester modifiable et transmissible.

### 5. L’anti-surpromesse
Cadris ne promet pas de tout faire.
Cadris promet de faire tenir le projet.

## Formulations recommandées
- cadre clair
- avancer sans dériver
- ce qui fait foi
- source de vérité
- mémoire de décision
- base exploitable
- projet tenable
- garder la cohérence
- projet transmissible
- faire évoluer sans se reperdre

## Formulations à éviter
- app builder
- générateur de documentation
- copilote total
- build autonome
- productivité infinie
- plus besoin de réfléchir
- ship plus vite comme promesse centrale
- tout faire de A à Z
- payez au run
- wrapper IA
