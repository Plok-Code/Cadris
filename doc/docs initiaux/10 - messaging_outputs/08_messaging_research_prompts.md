# 08_messaging_research_prompts

## Prompt 1 — Vocabulaire du marché
Analyse le vocabulaire dominant utilisé par les outils et services qui s’adressent aux builders SaaS ou aux utilisateurs de code assisté par IA.
Identifie :
- les termes les plus fréquents ;
- les oppositions récurrentes ;
- les expressions qui tournent autour de vitesse, autonomie, génération, contrôle, structure et fiabilité ;
- les mots qui pourraient aider Cadris à parler d’un “cadre clair” sans tomber dans le jargon.
Objectif :
repérer le vocabulaire déjà saturé et les formulations encore sous-exploitées pour parler de cohérence, continuité et tenue du projet.

## Prompt 2 — Promesses saturées
Étudie les promesses les plus répétées sur ce marché :
- idea to app ;
- build faster ;
- ship in minutes ;
- AI agent builder ;
- full-stack automation ;
- documentation automatique.
Pour chacune :
- explique pourquoi la promesse est saturée ;
- indique le risque de confusion si Cadris s’en approche trop ;
- propose une alternative de formulation plus crédible et plus distinctive pour Cadris.
Objectif :
éviter que le messaging de Cadris replonge dans les promesses banales ou bruyantes du marché.

## Prompt 3 — Formulations concurrentes à éviter
Recense les formulations utilisées par les concurrents directs, indirects et alternatives proches sur les territoires suivants :
- app builder IA ;
- copilote de code ;
- outil de documentation produit ;
- couche de collaboration / mémoire ;
- accompagnement produit ou technique.
Identifie :
- les mots ou structures de phrase à éviter car trop proches ;
- les formulations trop génériques ou trop ambiguës ;
- les éléments de langage qui feraient glisser Cadris vers “builder de plus”, “doc tool” ou “service déguisé”.
Objectif :
protéger la singularité du messaging et éviter les confusions de catégorie.

## Prompt 4 — Preuves utilisées par le marché
Observe comment les acteurs du marché prouvent leur valeur :
- démonstration produit ;
- avant / après ;
- capture d’usage ;
- cas d’usage ;
- promesse de gain ;
- argument technique ;
- crédibilité d’artefact.
Identifie :
- les preuves les plus fréquentes ;
- les preuves les plus faibles ;
- les preuves les plus crédibles sur un territoire de cohérence, structure et transmission ;
- les manières dont Cadris pourrait montrer qu’un projet “tient” mieux avec un cadre clair.
Objectif :
renforcer les points de crédibilité du messaging sans sur-promesse.

## Prompt 5 — Risques de flou du messaging
À partir du territoire de marque suivant :
- cadre clair ;
- anti-dérive ;
- source de vérité ;
- mémoire de décision ;
- projet transmissible ;
évalue les principaux risques de flou ou de mauvaise lecture.
Cherche notamment :
- à quel moment le discours bascule vers la documentation ;
- à quel moment il devient trop abstrait ;
- à quel moment “vibe codé” devient trop niche ;
- à quel moment la promesse ressemble à du conseil ;
- à quel moment l’offre devient difficile à comprendre.
Objectif :
identifier les zones de flou du messaging avant industrialisation sur landing, onboarding ou sales.

## Utilisation recommandée
Utiliser ces prompts pour :
- préparer une itération de landing ;
- comparer plusieurs variantes de hook ;
- renforcer les preuves ;
- affiner le wedge d’acquisition sans durcir excessivement la marque.
