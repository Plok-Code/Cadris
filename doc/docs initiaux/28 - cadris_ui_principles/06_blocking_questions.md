# 06_blocking_questions

## Questions bloquantes restantes

### BQ-01 - Quelle surface doit dominer dans la mission room V1 ?
- Pourquoi cela bloque : c'est la decision qui structure le plus fortement le futur wireframing.
- Ce qu'il faut obtenir pour avancer : arbitrer entre :
  - `hub synthese d'abord`
  - `bloc actif d'abord`
  - `feed mission plus central`
- Hypothese temporaire : `hub synthese d'abord`, puis acces direct au bloc actif.

### BQ-02 - Quel niveau d'edition directe doit exister dans les blocs ?
- Pourquoi cela bloque : les composants, les etats, les CTA et la densite changent beaucoup selon que l'utilisateur valide un contenu structure, ou edite inline de facon plus libre.
- Ce qu'il faut obtenir pour avancer : une decision entre :
  - `dialogue + validation structuree`
  - `edition inline legere`
  - `edition documentaire plus directe`
- Hypothese temporaire : `dialogue + validation structuree`, avec edition legere sur certains champs.

### BQ-03 - Que couvre le mobile en V1 ?
- Pourquoi cela bloque : la navigation, la densite et la profondeur d'ecran ne sont pas les memes si le mobile doit produire, relire ou seulement consulter.
- Ce qu'il faut obtenir pour avancer : une decision entre :
  - `consultation seulement`
  - `consultation + reponses`
  - `production complete`
- Hypothese temporaire : `consultation + reponses`.

### BQ-04 - Souhaite-t-on un dark mode des la V1 ?
- Pourquoi cela bloque : il n'empeche pas de fixer les principes, mais il change la finalisation des couleurs, contrastes, etats et assets.
- Ce qu'il faut obtenir pour avancer : une decision entre `non en V1`, `plus tard`, ou `oui des maintenant`.
- Hypothese temporaire : `plus tard`.

## Pourquoi ces points bloquent

- BQ-01 conditionne la structure des ecrans centraux.
- BQ-02 conditionne la forme des composants critiques de production.
- BQ-03 conditionne la declinaison responsive des principes.
- BQ-04 conditionne la finalisation visuelle haute fidelite.

## Priorite recommandee

1. arbitrer la mission room ;
2. arbitrer le mode d'edition ;
3. clarifier le mobile V1 ;
4. confirmer ou non le dark mode.

- Bloquant pour definir les principes UI : non.
- Bloquant pour detailller des wireframes solides : oui.
