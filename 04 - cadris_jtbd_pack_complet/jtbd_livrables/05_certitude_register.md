# Registre de certitude

## Confirmé
- La cible ne cherche pas principalement à coder plus vite.
- Le besoin central est de ne pas perdre le contrôle quand le prototype devient un vrai produit.
- Le pain dominant est la perte de continuité décisionnelle et structurelle.
- Les symptômes récurrents sont : régressions, confusion, peur de toucher au projet, besoin de rebuild, difficulté de transmission.
- Le passage builder → GitHub / Cursor / workflow plus contrôlé est un motif crédible et répété.
- Les outcomes les plus importants tournent autour de : reprendre, comprendre, modifier sans casser, transmettre et garder la vitesse sans chaos.
- Le wedge initial le plus crédible concerne des SaaS métier simples à moyens, structurables, avec logique claire de données, rôles et workflows.

## Hypothèses de travail
- La douleur d’achat s’exprime mieux comme **contrôle + continuité** que comme **documentation**.
- Impact :
  - oriente le produit vers un système de structuration et de gouvernance légère, pas vers un simple générateur de docs.
- Pourquoi cette hypothèse a été retenue :
  - les signaux montrent que la documentation seule n’est pas la fin recherchée ; elle semble être un moyen au service de la maîtrise.

- L’entrée marché la plus crédible est le segment des builders déjà passés par Lovable, v0, Replit ou outils voisins, puis bloqués au moment où le projet grossit.
- Impact :
  - aide à formuler un récit d’entrée clair ;
  - facilite l’identification de la douleur.
- Pourquoi cette hypothèse a été retenue :
  - le motif “wow initial puis plafond structurel” revient fortement.

- Le périmètre produit le plus crédible pour une V1 est **avant-code + continuité légère pendant le build**.
- Impact :
  - positionnement plus réaliste ;
  - réduction du risque de sur-promesse.
- Pourquoi cette hypothèse a été retenue :
  - les signaux montrent que le mur n’est pas seulement avant le code ; il apparaît aussi pendant la phase d’évolution du produit.

## Inconnus
- Le déclencheur monétaire exact de l’achat.
- Pourquoi ce point reste inconnu :
  - les signaux publics montrent la douleur, mais pas précisément quand la dépense devient évidente.
- Quel impact potentiel :
  - fort impact sur le packaging, la vente et le moment de conversion.

- La forme de solution la plus achetable.
- Pourquoi ce point reste inconnu :
  - on ne sait pas encore si la cible paierait surtout pour un produit self-serve, un audit, une migration assistée, un copilote de structuration ou un hybride.
- Quel impact potentiel :
  - fort impact sur l’offre, le pricing et le go-to-market.

- Le sous-déclencheur le plus douloureux.
- Pourquoi ce point reste inconnu :
  - plusieurs points de rupture apparaissent : backend, auth, SEO, data, régressions, handoff, perte de confiance.
- Quel impact potentiel :
  - impact moyen à fort sur le message d’entrée.

## Bloquants
- Absence d’entretiens directs avec des builders correspondant exactement au moment “proto déjà crédible, mais pas encore structuré”.
- Pourquoi c'est bloquant :
  - sans entretiens directs, la hiérarchie fine des déclencheurs d’achat reste approximative.
- Ce qu'il faut obtenir pour débloquer :
  - 5 à 10 entretiens ciblés ou un corpus d’échanges directs comparables.

- Absence de validation nette sur le format acheté.
- Pourquoi c'est bloquant :
  - il est risqué de passer au positionnement détaillé sans savoir si la cible achète une couche de cadrage, une aide de migration ou une gouvernance du build.
- Ce qu'il faut obtenir pour débloquer :
  - réponses terrain sur ce que la cible paierait en premier.

## Statut de transmission
- Transmission autorisée : Oui sous hypothèses
- Raison :
  - le noyau JTBD est assez solide pour transmettre vers l’étape suivante, mais les choix de packaging, de timing d’achat et de forme d’offre doivent encore être validés.

## Handoff au GPT suivant
- Le cœur du problème n’est pas la vitesse de génération de code.
- Le cœur du problème est la transition fragile entre prototype rapide et produit durable.
- Toute proposition suivante devra vérifier en priorité :
  - le déclencheur d’achat réel ;
  - la promesse la plus achetable ;
  - le format de solution le plus crédible.
