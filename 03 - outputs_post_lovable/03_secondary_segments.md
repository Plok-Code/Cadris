# Segments secondaires

## Segment secondaire 1 — Sous-segment “post-Lovable / post-builder IA bloqué”
### Description
Utilisateurs qui ont déjà goûté à la vitesse d’un builder IA, mais qui sentent maintenant un plafond de complexité, de personnalisation ou de maintenabilité.

### Intérêt futur
- comparaison produit très concrète ;
- douleur immédiatement reconnaissable ;
- bon angle de wedge narratif : “tu as lancé vite, maintenant il faut structurer proprement”.

### Raisons de ne pas le formaliser comme segment unique principal tout de suite
- besoin de validation terrain plus précise ;
- risque de dépendre trop fortement d’un concurrent donné ;
- tous les utilisateurs bloqués ne partent pas forcément pour les mêmes raisons.

---

## Segment secondaire 2 — Fondateur produit ambitieux mais peu structuré techniquement
### Description
Profil fort sur la vision, le cadrage fonctionnel et l’ambition produit, mais faible sur l’architecture, les conventions, la documentation exécutable et l’ordre de build.

### Intérêt futur
- bonne sensibilité à la valeur de cadrage ;
- potentiel de bouche-à-oreille si le livrable est clair ;
- peut devenir un segment très rentable si le produit aide vraiment à transformer une vision en système exécutable.

### Raisons de ne pas le prioriser maintenant
- segment encore large ;
- risque d’étendre le produit vers du product strategy tooling ;
- moins net que le cas “j’ai déjà bricolé et maintenant je bloque”.

---

## Segment secondaire 3 — PM / designer / operator très hands-on avec IA
### Description
Utilisateurs qui pilotent déjà des outils IA pour matérialiser rapidement des flux, interfaces ou logiques, mais qui ne maîtrisent pas la montée en complexité d’un vrai SaaS.

### Intérêt futur
- comprend la valeur d’un dossier canonique ;
- peut utiliser la documentation pour collaborer avec développeurs ou agents ;
- segment potentiellement fertile pour la diffusion.

### Raisons de ne pas le prioriser maintenant
- rôles plus hétérogènes ;
- douleur parfois moins aiguë si le projet n’est pas porté en direct ;
- peut nécessiter des besoins de collaboration multi-rôles plus tôt que souhaité.

---

## Segment secondaire 4 — Indie hacker déjà semi-outillé
### Description
Builder déjà assez autonome techniquement, avec repo, règles, prompts structurés et habitudes de build plus solides.

### Intérêt futur
- segment prescripteur ;
- peut valoriser des fonctionnalités profondes de structuration avancée ;
- bon test de robustesse produit à terme.

### Raisons de ne pas le prioriser maintenant
- alternatives maison déjà présentes ;
- exigence de profondeur plus élevée ;
- plus difficile à convaincre sur une V1.

---

## Conclusion
Les segments secondaires les plus plausibles gravitent tous autour d’un même noyau :
**des gens déjà convaincus par la puissance de l’IA pour construire, mais pas encore capables de tenir un gros projet dans la durée.**

Ce noyau peut ensuite être élargi selon :
- le rôle exact ;
- l’outil de départ ;
- le niveau de douleur ;
- le besoin de collaboration.
