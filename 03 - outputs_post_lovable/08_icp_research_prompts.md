# Prompts de recherche ICP

## 1. Prompt de segmentation marché
Analyse le marché des personnes qui utilisent déjà un peu les outils IA pour construire des apps, mais qui veulent maintenant passer d’un prototype ou d’une petite app à un **SaaS complet avec vraie architecture long terme**.

Je veux que tu segmentes ce marché selon :
- niveau réel de maîtrise des outils IA de code ;
- type de projet porté ;
- expérience passée : simple proto vs projet déjà structuré ;
- degré de douleur quand la complexité augmente ;
- besoin de documentation exploitable ;
- capacité à payer ;
- autonomie d’achat ;
- proximité avec des alternatives déjà suffisantes.

Pour chaque segment, précise :
- qui est l’utilisateur ;
- qui est l’acheteur ;
- qui est le décideur ;
- quelle douleur domine ;
- quel niveau d’urgence existe ;
- quels outils ils utilisent déjà ;
- si le segment paraît accessible pour une V1.

Termine par :
- segment prioritaire recommandé ;
- segments secondaires ;
- anti-cible ;
- degré de confiance.

---

## 2. Prompt d’identification des décideurs et acheteurs
Dans le marché des builders IA qui veulent transformer un proto en vrai SaaS, identifie les rôles réels impliqués dans l’achat et l’usage.

Je veux que tu distingues :
- utilisateur principal ;
- acheteur ;
- décideur ;
- influenceurs ;
- prescripteurs éventuels.

Fais-le pour ces profils :
- solo founder ;
- indie builder ;
- ex-utilisateur Lovable ou builder IA bloqué ;
- PM / designer hands-on ;
- petite équipe très early.

Pour chaque profil, précise :
- qui ressent la douleur ;
- qui paie ;
- qui bloque ou valide la décision ;
- quel vocabulaire métier ils utilisent réellement.

---

## 3. Prompt sur la maturité et la capacité à payer
Étudie quels segments du marché “proto IA -> vrai SaaS” ont la plus forte capacité à payer pour un produit qui structure, documente et guide la construction d’un projet plus ambitieux.

Je veux que tu compares :
- novices complets ;
- builders déjà capables de sortir une petite app ;
- ex-utilisateurs Lovable / v0 / builders rapides ;
- fondateurs produits peu structurés techniquement ;
- indie hackers déjà semi-outillés.

Pour chaque segment, précise :
- niveau de conscience du problème ;
- intensité de la douleur ;
- capacité à payer ;
- probabilité d’achat en self-serve ;
- probabilité d’adhésion à un produit guidé ;
- objections probables.

Conclue par :
- segment avec le meilleur ratio douleur / compréhension / paiement ;
- segment à éviter au départ.

---

## 4. Prompt sur les signaux d’urgence du segment
Analyse à quel moment précis un builder IA ressent une urgence assez forte pour acheter une solution de structuration et de documentation.

Je veux que tu identifies les moments possibles :
- avant de coder ;
- après un premier proto ;
- quand l’application commence à devenir complexe ;
- quand les prompts deviennent incohérents ;
- quand il envisage une migration depuis Lovable ou un builder rapide ;
- quand il craint un rebuild.

Pour chaque moment, précise :
- niveau d’urgence ;
- symptôme visible ;
- langage utilisé par l’utilisateur ;
- déclencheur d’achat probable ;
- type de promesse produit qui résonne le mieux.

Termine par une recommandation du meilleur moment d’entrée.

---

## 5. Prompt sur les segments à exclure
Liste les segments à exclure pour la V1 d’un produit qui aide à passer d’un proto IA bricolé à un vrai SaaS structuré.

Je veux une anti-cible argumentée.

Analyse notamment :
- novices complets ;
- utilisateurs qui veulent seulement générer une app vite ;
- développeurs seniors déjà très structurés ;
- équipes déjà outillées ;
- agences multi-clients ;
- projets trop réglementés ou trop complexes.

Pour chaque segment exclu, précise :
- pourquoi la douleur est mal alignée ;
- pourquoi la perception de valeur sera faible ;
- quel risque produit ou go-to-market cela crée ;
- à quel moment ce segment pourrait devenir intéressant plus tard.
