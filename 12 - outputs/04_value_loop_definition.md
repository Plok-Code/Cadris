# 04_value_loop_definition

## Point d’entrée
Le builder arrive avec un proto ou un projet déjà réel, assez concret pour être structuré, repris, modifié ou transmis.

## Action clé utilisateur
Il clarifie ce qui doit faire foi :
- objectif du produit ;
- périmètre ;
- logique centrale ;
- décisions importantes ;
- structure exploitable pour la suite.

## Résultat visible
Le produit génère et maintient :
- un dossier canonique exportable ;
- une mémoire de décision claire ;
- une base commune pour reprendre, modifier ou transmettre le projet.

## Moment où la valeur est perçue
La valeur est perçue quand l’utilisateur peut :
- comprendre rapidement l’état réel du projet ;
- reprendre après interruption ;
- préparer un changement sans se reperdre ;
- transmettre à quelqu’un d’autre sans recontextualiser tout le projet.

## Boucle minimale recommandée
1. **Projet entrant**
2. **Structuration canonique**
3. **Usage concret pour reprise / modification / transmission**
4. **Retour d’une nouvelle décision importante**
5. **Mise à jour légère de la base**
6. **Projet qui tient mieux dans le temps**

## Boucle de valeur résumée
**Projet flou -> source de vérité -> reprise/modification/transmission plus sûres -> mise à jour légère -> projet plus tenable**
